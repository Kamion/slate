    Introduction

    Welcome to the PathSquad API! Here you will find all the endpoints with their requests and responses, which will allow you to harness the power of PathSquad routing in your own apps.

    You can view code examples in the dark area to the right.

    To register for PathSquad services, please visit https://maps.pathsquad.com.

    Authentication

    To authorize, use this code:

    # With shell, you can just pass the correct header with each request
curl "https://api.pathsquad.com"
  -H "Authorization: Bearer [developer_token]"

    Make sure to replace [developer_token] with your API key.

    PathSquad uses an authorization header with a developer key to allow access to the API. You can find your API key in your profile page.

    PathSquad expects for the API key to be included in all API requests to the server in a header that looks like the following:

    Authorization: Bearer [developer_token]

    You must replace [developer_token] with your personal API key.

    The route response can be used as is in the MapBox mobile SDK. To decode the the polyline, please keep in mind that you have to use six digits as opposed to five stated in the Google documentation. You can find more info and examples here. The predicted stops are suggestions based on our propietary algorithms which aim to guide the drivers to the most efficient stops for their particular route.

    HTTP Request

    POST https://api.pathsquad.com/route

    Request Body

    Provide the stops in the order you want them to appear in the route. The depart_at property is taken into account when calculating the ETA for the route. You can leave it empty and the service will take the current local time based on departure location as a starting time. The string format is RFC3339.

    Get mileage breakdown per state

    curl "https://api.pathsquad.com/ifta/<ID>"
  -H "Authorization: Bearer [developer_token]"

    The above command returns JSON structured like this:

    {
  "IL": 42357.37650526342,
  "IN": 252314.33842087357,
  "NJ": 111204.47424270134,
  "NY": 4184.132985528076,
  "OH": 381309.4708496073,
  "PA": 500267.60900097404
}

    This endpoint returns the mileage breakdown per state. The URL parameter is the property route_id from the route request. The route ID is available for 10 minutes after the route request is executed. The distances in the response are in meters.

    HTTP Request

    GET https://api.pathsquad.com/ifta/<ID>

    URL Parameters

    Parameter Description
    ID The route ID, generated by route request

    This endpoint returns the weather forecast along the route. The URL parameter is the property route_id from the route request. The route ID is available for 10 minutes after the route request is executed. Weather forecast is retrieved for the relative time and location the truck is estimated to be at.

    HTTP Request

    GET https://api.pathsquad.com/weather/<ID>

    URL Parameters

    Parameter Description
    ID The route ID, generated by route request

    Errors

    The PathSquad API uses the following error codes:

    Error Code Meaning
    400 Bad Request -- Your request is invalid.
    401 Unauthorized -- Your API key is wrong.
    404 Not Found -- The specified endpoint could not be found.
    500 Internal Server Error -- We had a problem with our server. Try again later.
    503 Service Unavailable -- We're temporarily offline for maintenance. Please try again later.
